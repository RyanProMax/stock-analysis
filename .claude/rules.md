# 代码分析规则

## 核心原则
- 只做结构理解，不做过度推测
- 不尝试一次性分析整个项目
- 不记忆历史对话，所有结论必须写入 `.claude/context.md`
- 后续分析只能基于 `context.md` 中已有内容 + 本轮新增内容
- 如果信息不足，明确指出，不要脑补

## 每个新会话必须
1. 首先读取 `.claude/context.md` 了解已知内容
2. 分析后更新 `context.md`
3. 所有结论有据可依，拒绝猜测
4. **输出后检查中文字符是否乱码，如有则修正**
